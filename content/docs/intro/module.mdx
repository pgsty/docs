## Core Modules

Pigsty consists of multiple **modules**. The `PINE` stack: `PGSQL` / `INFRA` / `NODE` / `ETCD` are <span className="text-sky-700 font-black">ESSENTIAL</span> for self-hosting Postgres RDS service.

<Cards>
    <Card icon={<Database className="text-sky-600" />} title='PGSQL' href={"/docs/pgsql"}>
        HA PG Cluster with PITR, IaC, ACL, Monitor, and 420+ extensions
    </Card>
    <Card icon={<Landmark className="text-green-700" />} title='INFRA' href={"/docs/infra"}>
        Nginx, Repo, DNS, NTP, Prometheus and Grafana stack for Observability
    </Card>
    <Card icon={<CpuIcon className={"text-amber-500"} />} title='NODE' href={"/docs/node"}>
        Enroll nodes into the desired state and monitor it, and VIP, HAProxy
    </Card>
    <Card icon={<Settings className={"text-sky-400"} />} title='ETCD' href={"/docs/etcd"}>
        Reliable distributive consensus storage (DCS), empowering PGSQL HA
    </Card>
</Cards>

## Extra Modules

Pigsty also have some <span className="text-emerald-500 font-black">OPTIONAL</span> "Bonus" modules, which works well with PostgreSQL, and brings extra value to your data infrastructure.

<Cards>
    <Card icon={<Box className="text-red-800" />} title='MINIO' href={"/docs/minio"}>
        S3 compatible object storage compatible, optional backup storage
    </Card>
    <Card icon={<Layers className="text-red-400" />} title='REDIS' href={"/docs/redis"}>
        High-performance in-memory cache, optinal data structure server
    </Card>
    <Card icon={<Container className="text-blue-500" />} title='DOCKER' href={"/docs/docker"}>
        Container runtime, optional for running stateless app and tools
    </Card>
    <Card icon={<Leaf className="text-gray-500" />} title='FERRET' href={"/docs/ferret"}>
